# 预估算手续费使用方法

本文档整理自 **dw20pledge** 项目，说明如何通过无链 RPC 接口预估转账/合约调用手续费，便于在其他项目中复用。

---

## 1. 概述

预估手续费用于在用户提交质押（或类似转账）前，展示「预估手续费」并计算「共计支付 = 质押数量 + 手续费」。

核心流程：

1. 调用 RPC `query` 接口，`request_type` 为 `transfer_fee`
2. 传入发送方、接收方、代币符号、金额等参数
3. 从响应中读取 `fees[0]`（链上最小单位）
4. 用 `formatNearAmount` 转为可读数量，再 `Math.ceil` 向上取整展示

## 3. RPC 接口说明

### 3.1 请求地址

| 环境 | URL |
|------|-----|
| 开发（代理） | `/rpc` → 代理到 testnet |
| 生产 / 直连 | `https://rpc-testnet.chainlessdw20.com/` |

> 主网或其他环境请替换为对应 RPC 地址。

### 3.2 请求格式

**Method:** `POST`  
**Content-Type:** `application/json`  
**Headers:** 建议携带 `rpc: true`（与本项目保持一致）

**Body（JSON-RPC 2.0）：**

```json
{
  "jsonrpc": "2.0",
  "id": "dontcare",
  "method": "query",
  "params": {
    "request_type": "transfer_fee",
    "finality": "final",
    "account_id": "发送方账户ID",
    "receiver_id": "接收方账户ID或合约名",
    "symbol": "TDW20",
    "amount": "10000000000000000000000",
    "fee_symbol": ["TDW20"]
  }
}
```

### 3.3 参数说明

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `request_type` | string | 是 | 固定为 `"transfer_fee"` |
| `finality` | string | 是 | 固定为 `"final"` |
| `account_id` | string | 是 | 发送方账户 ID（当前用户） |
| `receiver_id` | string | 是 | 接收方 ID，质押场景为合约名，如 `dw20-staking-pool.contract` |
| `symbol` | string | 是 | 转账代币符号，如 `TDW20` |
| `amount` | string | 是 | **链上最小单位**字符串，需用 `parseNearAmount` 转换 |
| `fee_symbol` | string[] | 是 | 手续费代币列表，如 `["TDW20"]` |

### 3.4 响应结构（参考）

成功时，业务数据在 `result.result` 中（具体以实际 RPC 返回为准）：

```json
{
  "status": 1,
  "result": {
    "result": {
      "fees": ["123456789012345678901234"]
    }
  }
}
```

- `fees[0]`：第一笔手续费，为链上最小单位（可能是 `BigInt` 或字符串）
- 本项目取 `fees[0]` 作为预估手续费

## 4. 金额单位转换

无链沿用了 NEAR 风格的 24 位小数：

```typescript
import * as nearAPI from 'chainless-api-ts';
const { utils } = nearAPI;

// 用户输入 "10000" → 链上最小单位
const chainAmount = utils.format.parseNearAmount('10000');
// 例: "10000000000000000000000"

// 链上最小单位 → 可读数量
const humanAmount = utils.format.formatNearAmount('123456789012345678901234');
// 例: "0.000123..." 或整数形式
```

**注意：**

- 传给 RPC 的 `amount` 必须是 `parseNearAmount` 后的字符串
- 从 `fees[0]` 读出的值需用 `formatNearAmount` 再转为展示用数字

## 5. 核心实现代码

### 5.1 RPC 封装（可直接复制）

```typescript
import axios from 'axios';
import * as nearAPI from 'chainless-api-ts';

const { utils } = nearAPI;

const RPC_URL =
  process.env.NODE_ENV === 'development'
    ? '/rpc'
    : 'https://rpc-testnet.chainlessdw20.com/';

/**
 * 发起 RPC query 请求
 */
export async function rpcPostHttpQuest(params: Record<string, unknown>) {
  const response = await axios.post(
    RPC_URL,
    {
      jsonrpc: '2.0',
      id: 'dontcare',
      method: 'query',
      params,
    },
    {
      headers: { rpc: 'true' },
      timeout: 50000,
    }
  );

  const data = response.data;

  // 与本项目 axios 拦截器逻辑一致：status === 1 时返回 result.result
  if (data.status === 1) {
    return data.result;
  }
  return data;
}
```

### 5.2 预估手续费函数（可直接复制）

```typescript
export interface EstimateTransferFeeParams {
  /** 发送方账户 ID */
  accountId: string;
  /** 接收方 ID / 合约名 */
  receiverId: string;
  /** 代币符号，如 TDW20 */
  symbol: string;
  /** 用户输入的可读金额，如 "10000" */
  amount: string;
  /** 手续费代币，默认 ["TDW20"] */
  feeSymbol?: string[];
}

/**
 * 预估转账手续费
 * @returns 向上取整后的手续费（可读整数，如 1 表示 1 DW20）
 */
export async function estimateTransferFee(
  params: EstimateTransferFeeParams
): Promise<number> {
  const {
    accountId,
    receiverId,
    symbol,
    amount,
    feeSymbol = ['TDW20'],
  } = params;

  try {
    const result = await rpcPostHttpQuest({
      request_type: 'transfer_fee',
      finality: 'final',
      account_id: accountId,
      receiver_id: receiverId,
      symbol,
      amount: utils.format.parseNearAmount(amount),
      fee_symbol: feeSymbol,
    });

    const rawFee = result.result.fees[0];

    // 兼容 BigInt
    const feeString =
      typeof rawFee === 'bigint' ? rawFee.toString() : String(rawFee);

    const formatted = utils.format.formatNearAmount(feeString);
    const ceilVal = Math.ceil(parseFloat(formatted));

    return ceilVal;
  } catch (error) {
    console.error('Error calculating fee:', error);
    return 0;
  }
}
```

### 5.3 质押场景调用示例

```typescript
import Configs from '@/configs/contract'; // dw20_staking_pool: 'dw20-staking-pool.contract'

const fee = await estimateTransferFee({
  accountId: userId,
  receiverId: 'dw20-staking-pool.contract',
  symbol: 'TDW20',
  amount: '10000', // 用户输入的质押数量
});

console.log(`预估手续费: ${fee} DW20`);

// 共计支付
const stakingAmount = 10000;
const totalPay = stakingAmount + fee;
```

## 6. 在本项目中的使用场景

来源文件：`src/pages/index/component/StakingComponent.tsx`

### 6.1 输入变化时自动计算

当用户输入质押数量 ≥ 10,000 时，触发手续费预估：

```typescript
useEffect(() => {
  if (inputValue && Number(inputValue) >= 10000) {
    estimatedHandlingFee(inputValue);
  } else {
    setEstimatedFee('0');
  }
}, [inputValue]);
```

### 6.2 点击「全部」时的特殊逻辑

质押全部余额时，需先算手续费，再从余额中扣除，避免余额不足：

```typescript
// 全部质押
const resultFee = await estimatedHandlingFee(balanceVal + '');
const feeNumber = typeof resultFee === 'number' ? resultFee : Number(resultFee);
const val = balanceVal - Math.ceil(feeNumber); // 实际可质押数量
setInputValue(val.toString());
```

### 6.3 确认质押前校验余额

```typescript
const stakingAmount = parseFloat(inputValue);
const feeAmount = parseFloat(estimatedFee);
const totalAmount = stakingAmount + feeAmount;

if (totalAmount > balanceVal) {
  // 显示「可用余额不足」
} else {
  // 执行 submitBtn() 发起质押
}
```

### 6.4 与真实质押参数保持一致

预估时的 `receiver_id`、`symbol`、`amount` 应与实际质押交易一致：

```typescript
// 实际质押参数（submitBtn）
{
  receiverId: 'dw20-staking-pool.contract',
  sender_account_id: userId,
  actions: {
    method_name: 'deposit',
    args: {},
    max_gas: 300000000000000,
    amount: utils.format.parseNearAmount(inputValue),
    symbol: 'TDW20',
    fee_symbol: 'TDW20',
  }
}
```

## 8. 注意事项

1. **BigInt 兼容**：`fees[0]` 可能是 `BigInt`，必须先 `.toString()` 再交给 `formatNearAmount`。
2. **向上取整**：展示时使用 `Math.ceil(parseFloat(val))`，避免小数显示与余额校验偏差。
3. **参数一致**：`account_id`、`receiver_id`、`symbol`、`amount` 必须与最终链上交易一致，否则预估值不准。
4. **最小质押量**：本项目最小 10,000 DW20，低于此值不调用预估（可按业务调整）。
5. **错误兜底**：请求失败时返回 `0`，并在 UI 上区分「计算中」与「计算失败」更佳。
6. **环境切换**：生产 API 见 `.umirc.ts` 中 `REACT_APP_PUBLIC_APIURL`；RPC 需按 testnet/mainnet 分别配置。
